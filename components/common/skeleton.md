# Skeleton

> 로딩 상태에서 사용자에게 **예상되는 레이아웃 구조와 정보 밀도를 암시**하여 **지각적 안정감**을 제공하는 UI 패턴.

## 📍 환경

- [x] **Common (Admin + Service 공통)**
- [ ] Service
- [ ] Admin

## ⚠️ 컴포넌트 형태

> **Figma에 Component Set이 정의되지 않은 패턴 컴포넌트입니다.**
>
> 이 페이지는 **디자인 원칙과 시각적 정의만 제공**합니다. 실제 구현은 각 콘텐츠 영역에 맞춰 **레이아웃을 그대로 복제하여** 색만 바꾸는 방식으로 적용합니다.

## 🎯 사용 시점

**비동기 데이터를 불러오는 모든 화면**에서 콘텐츠가 표시되기 전에 노출합니다. 단순한 스피너보다 더 풍부한 정보(레이아웃 구조, 정보 양)를 사용자에게 미리 제공할 수 있습니다.

### 이럴 때 사용
- 페이지 첫 진입 시 (콘텐츠 fetch 중)
- 무한 스크롤 추가 로딩
- 이미지 로딩 중
- 카드/리스트 등 반복 콘텐츠 로딩
- 새로고침 (pull-to-refresh)

### 사용하지 말아야 할 때
- **로딩이 필요 없는 즉시 표시 가능한 영역** (Figma 가이드 원문)
- 짧은 액션의 결과 대기 (1초 미만) → 차라리 자연스러운 transition
- 단일 버튼 클릭 후 처리 → **Spinner**나 **Toast**
- 백그라운드 작업 → 사용자에게 알림 X

### Skeleton vs Spinner vs Progress

| | **Skeleton** | **Spinner** | **Progress Bar** |
|---|---|---|---|
| 정보량 | **레이아웃 + 콘텐츠 양 암시** | 로딩 중임만 | 진행도 % |
| 상황 | 콘텐츠 로딩 (시간 불명) | 짧은 액션 처리 | 명확한 작업 진행 |
| 사용자 인지 | "곧 콘텐츠 나오겠구나" | "기다려야 함" | "X% 남음" |

## 🧩 구조

Skeleton은 **실제 콘텐츠 구조를 그대로 복제하고 색만 바꾸는** 방식으로 만들어집니다.

```
[원본 콘텐츠]                    [Skeleton]
┌──────────┐                    ┌──────────┐
│  🖼️ 이미지  │           →        │  ░░░░░░  │  (gray 6 #DFDFDF)
├──────────┤                    ├──────────┤
│  제목       │                    │  ▒▒▒▒▒    │  (radius 13, #F5F4F3)
│  설명       │                    │  ▒▒▒▒▒▒▒  │  (radius 13, #F5F4F3)
└──────────┘                    └──────────┘
```

### 핵심 원칙

> **"실제 콘텐츠 구조를 반영하여 사용자가 콘텐츠를 예상하도록 돕습니다.**
> **의미없는 장식요소를 포함하지 않습니다."**

- ✅ 실제 콘텐츠와 **같은 위치, 비슷한 크기**로 배치
- ✅ 같은 컴포넌트 그룹은 **반복 횟수도 비슷하게** (예: 카드 3개 → skeleton 3개)
- ❌ 의미 없는 장식 요소 추가 X
- ❌ 실제와 다른 레이아웃 X

## 🎨 사용 토큰

### 영역별 색상 (가장 중요)

| 영역 | Color | 사용 |
|---|---|---|
| **이미지 / Thumbnail** | **`#DFDFDF` (gray 6)** | 이미지 영역만 별도 강조 |
| 텍스트 / 일반 영역 | `#F5F4F3` | 라인, 카드 영역 등 |
| 버튼 placeholder | `#ECECEC` ~ `#EFEFEF` | 약간 진한 회색 |
| Helper text 영역 | `#EFEFEF` | 작은 라벨 |

> 🔑 **이미지는 의도적으로 더 진한 회색** — 콘텐츠 비중이 가장 크다는 것을 시각적으로 암시 (Figma Description 원문)

### Radius 패턴

| 영역 | Radius | 비고 |
|---|---|---|
| **텍스트 라인 (높이 20~26)** | **13~14** | **거의 캡슐** 느낌 |
| Avatar 원형 (32×32) | 16 | 완전 원 |
| Avatar 원형 (24×24) | 13 | 완전 원 |
| 이미지 컨테이너 | 6 / 12 / 20 | **실제 콘텐츠 radius와 동일하게** |
| 버튼 placeholder | 16 | Solid 버튼과 동일 |
| 카드 외곽 | 12 / 20 | 실제 카드와 동일 |

> 💡 **텍스트 라인이 거의 캡슐 모양**인 게 특징 — 단순 사각형보다 부드러운 인상

### 크기 가이드

#### 텍스트 라인 (높이별)
| 높이 | 매핑 |
|---|---|
| 20 | 13~14px Medium 텍스트 1줄 |
| 22 | 15px Medium 텍스트 |
| 24 | 16px Medium 텍스트 |
| 26 | chip, badge |
| 28 | 큰 chip / 액션 버튼 |

#### 너비 패턴 (다양성)
- 100% (그리드 가로 전체)
- 80~85% (자연스러운 텍스트 끝)
- 50~60% (마지막 줄 짧게)
- 30~40% (Author/Tag 류)

> 💡 **너비를 다양하게** 사용해 실제 콘텐츠 느낌 살리기 (모두 100%면 어색)

## 📖 사용 가이드

### Figma 가이드 원문 정리

> **원칙:** 로딩 상태에서 사용자에게 **예상되는 레이아웃 구조와 정보 밀도를 암시**하여 **지각적 안정감을 제공**하기 위한 UI 패턴
>
> **활용:** 실제 콘텐츠 구조를 반영하여 사용자가 콘텐츠를 예상하도록 돕습니다. **의미없는 장식요소를 포함하지 않습니다.**
>
> **Description 1:** 다양한 콘텐츠의 높이와 가로길이에 맞춰 제공합니다. **로딩이 필요없는 기본 요소에는 스켈레톤을 제공하지 않습니다.**
>
> **Description 2:** **이미지 영역은 gray 6을 사용합니다.**

### Skeleton 작성 4단계

```
1. 실제 콘텐츠의 레이아웃을 그대로 복제
   ↓
2. 모든 텍스트 → 회색 박스 (#F5F4F3, radius 13)
   ↓
3. 모든 이미지 → 더 진한 회색 박스 (#DFDFDF, 같은 radius)
   ↓
4. 아이콘/숫자 → "--" 회색 텍스트 (#DFDFDF) 또는 회색 박스
```

### 구현 예시 — Post Card

```
[원본]
┌────────────────────────────────┐
│ 🟣 Avatar  Username     1m ago  │
│ ┌──────────────────────────┐    │
│ │     이미지 (carousel)      │    │

│ 게시물 제목                        │
│ 게시물 본문 한 줄...                │
│ 본문 두 번째 줄                    │
│ #태그                             │
│ ❤ 23  💬 5                      │
└────────────────────────────────┘

[Skeleton]
┌────────────────────────────────┐
│ ⚪ ▒▒▒▒                          │
│ ┌──────────────────────────┐    │
│ │ ░░░░░░░░░░░░░░░░░░░ (gray 6) │    │
│ └──────────────────────────┘    │
│ ▒▒▒▒▒▒▒▒                        │ ← 50%
│ ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒              │ ← 95%
│ ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒                  │ ← 80%
│ ▒▒▒▒▒                            │ ← 30%

└────────────────────────────────┘
```

### 무엇을 Skeleton으로 만들지 결정

#### Skeleton 적용
- ✅ 비동기 데이터 (서버에서 가져오는 콘텐츠)
- ✅ 이미지 (특히 사용자 업로드 이미지)
- ✅ 사용자 정보 (Avatar, Username)
- ✅ 본문 텍스트, 제목

#### Skeleton 적용 X
- ❌ 정적 UI (Tab Bar, Top Navigation 외곽 — 단, 페이지 타이틀은 skeleton)
- ❌ 즉시 사용 가능한 버튼 (정적인 시스템 액션)
- ❌ 로컬 state 기반 UI (켜고 끄는 토글 등)

### 반복 패턴

리스트 형태 콘텐츠는 **3~5개의 skeleton을 미리 그려두는 것** 권장:
- **3개**: 한 화면에 보이는 갯수
- **5개**: 약간의 스크롤 여유까지 포함

### 애니메이션 (구현 시)

Figma 정의에는 없지만 권장:
- **Shimmer**: 좌→우로 밝은 그라디언트가 흐르는 효과
- **Pulse**: opacity 0.5 ↔ 1 부드러운 깜빡임
- **속도**: 1.5~2초 주기 (너무 빠르면 시선 분산)

### 데이터 도착 시 전환

```
Skeleton → 실제 콘텐츠
   ↓
페이드 인/아웃 (200ms)
또는
같은 위치에 자연스럽게 swap
```

> 💡 갑작스러운 변화보다 부드러운 전환

## 🔄 State 처리

### 흐름도

```
[페이지 진입]
   ↓
Skeleton 표시 시작
   ↓
   API 호출 / 데이터 로딩
   ├ 데이터 도착 → Skeleton 사라지며 콘텐츠 표시
   ├ 에러 발생 → Skeleton 사라지고 Error 화면
   └ 빈 데이터 → Skeleton 사라지고 Empty state
```

### 무한 스크롤

```
화면 끝 도달
   ↓
하단에 추가 Skeleton 노출 (1~3개)
   ↓
새 데이터 도착 → Skeleton이 콘텐츠로 swap
```

## 🚫 금지 사항

- ❌ **콘텐츠와 다른 레이아웃 사용** — 실제 콘텐츠 구조 그대로 반영
- ❌ **로딩이 필요 없는 영역에 적용** — "기본 요소에는 스켈레톤 제공 X"
- ❌ **이미지 영역에 일반 회색 사용** — `#DFDFDF` (gray 6) 고정
- ❌ **모든 너비 100%** — 단조로움, 자연스럽게 다양화
- ❌ **장식 요소 추가** — "의미없는 장식요소를 포함하지 않습니다"
- ❌ **너무 오래 노출** — 5초 이상은 다른 UX 검토 (스피너 + 메시지)
- ❌ **Skeleton 깜빡임 없음** — shimmer/pulse 애니메이션 권장 (정적이면 멈춘 듯한 인상)
- ❌ **데이터 도착 시 깜빡임** — 자연스러운 전환 필요
- ❌ **Spinner와 동시 사용** — 한 화면 한 가지만
- ❌ **다른 색상 토큰 사용** — `#F5F4F3` / `#DFDFDF` 외 X
