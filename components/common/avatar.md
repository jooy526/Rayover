# Avatar

> 사용자 또는 기업의 **시각적 표현**. 아이콘, 로고, 사진 등 다양한 이미지 형식으로 표현됩니다.

## 📍 환경

- [x] **Common (Admin + Service 공통)**
- [ ] Service
- [ ] Admin

## 🎯 사용 시점

- 사용자 프로필 표시
- 기업/브랜드 로고 표시
- 댓글 작성자, 게시글 작성자 등 **주체(identity)** 시각화
- 팀/그룹 멤버 목록 표시

### 사용하지 말아야 할 때
- 단순 아이콘 용도 → **Icon** 사용
- 상태 표시 용도 → **Badge** 사용

## 🎨 Variants

### size (6단계)

| 이름 | 크기 | 주요 용도 |
|---|---|---|
| `XSmall` | 20 × 20 | 리스트 셀 내 보조 표시, 밀집된 영역 |
| `Small` | 24 × 24 | 댓글, 작은 리스트 |
| `Medium` | 32 × 32 | 일반 리스트 아이템 (기본 선택) |
| `Large` | 40 × 40 | 카드, 헤더 |
| `XLarge` | 48 × 48 | 프로필 요약, 강조 영역 |
| `XXLarge` | 56 × 56 | 프로필 페이지, 상세 영역 |

### style (2가지)

- **empty** — 플레이스홀더 (데이터 없음)
- **img** — 이미지 적용

## 🎯 사용 토큰

- **모양**: `radius/full` (원형) — 모든 사이즈 공통
- **empty 배경**: Gray 계열 (`label/assistive` 근처)
- **empty 아이콘**: 회색 톤의 사람 실루엣 (플레이스홀더 자산)
- **img 배경**: 이미지 자체
- **외곽선(테두리)**: 없음 (필요 시 group 컨테이너에서 추가)

## 📖 사용 가이드

### 플레이스홀더 사용 원칙
- 사용자 이미지 데이터가 **없거나 로딩 실패** 시 `style=empty` 사용
- 로딩 중에도 `style=empty` → 이미지 로드 후 `style=img` 전환

### 사이즈 선택 기준
- **컨테이너 크기에 비례**해서 선택
- 리스트 내에서 같은 depth면 **사이즈 통일**
- 정보 계층:
  - 메인 주체(프로필 페이지) → `XXLarge` / `XLarge`
  - 카드 대표 → `Large`
  - 리스트 기본 → `Medium`
  - 조밀한 리스트/댓글 → `Small`
  - 메타 정보 → `XSmall`

### 여러 Avatar 나란히 배치 (Avatar Group)
- 사이즈는 **반드시 동일**하게
- 가로 배치 시 약간씩 겹쳐 그룹감 표현 가능
- N명 이상일 때 "+N" 표시 Avatar로 축약 권장

### Child items (샘플 이미지)
Figma에 테스트용 `Avatar 1~10` 샘플 이미지가 포함되어 있음.
**프로토타이핑/디자인 검토 용도로만** 사용하고, 실제 프로덕션에서는 사용하지 말 것.

## 🔄 State 처리

- **Default (empty)**: 플레이스홀더 표시
- **Default (img)**: 이미지 표시
- **Loading**: `empty` 상태로 표시
- **Error (이미지 로드 실패)**: `empty`로 fallback
- **Pressed (클릭 가능한 경우)**: 오버레이 효과 또는 링 표시

## 🚫 금지 사항

- ❌ **사각형 Avatar 사용** — 시스템은 원형만 지원. 사각형 이미지가 필요하면 Thumbnail/Image 별도 사용
- ❌ **임의 크기 사용** — 정의된 6단계 외 크기 금지
- ❌ **외곽선/그림자 임의 추가** — 일관성 유지
- ❌ **텍스트 이니셜 Avatar** — 현재 시스템엔 정의되지 않음 (필요시 디자인 팀에 요청)
- ❌ **Child items 이미지 프로덕션 사용** — 샘플/목업용임
