# Tooltip

> 요소의 **짧은 부가설명**이 필요할 때 사용하는 호버형 정보 컴포넌트. 사용자가 마우스를 올리거나 포커스했을 때 떠올라 간단한 안내를 제공합니다.

## 📍 환경

- [x] **Common (Admin + Service 공통)**
- [ ] Service
- [ ] Admin

## 🎯 사용 시점

**짧은 부가 설명**이 필요할 때, 요소 위에 살짝 떠올리는 정보 표시용입니다. 콘텐츠를 가리지 않으면서도 필요한 정보를 자연스럽게 제공합니다.

### 이럴 때 사용
- 아이콘 버튼의 의미 설명 ("저장", "삭제" 등)
- 짧은 키워드/단축키 안내
- 비활성 요소의 사유 표시 ("권한 없음")
- 말줄임된 긴 텍스트의 전체 내용
- 작은 영역 안 약어/줄임말의 풀이

### 사용하지 말아야 할 때
- 사용자 액션(닫기/확인 등)이 필요한 정보 → **Popover**
- 페이지 인라인 안내 → **Callout**
- 일시적 결과 알림 → **Toast**
- 옵션 선택 → **Pulldown menu** / **Dropdown**
- 두 줄 이상 긴 설명 → **Popover** (Tooltip은 한 줄)

### Tooltip vs Popover (가장 헷갈리는 비교)

| | **Tooltip** | **Popover** |
|---|---|---|
| 트리거 | **호버 / 포커스** | 클릭 |
| 사이즈 | 작음 (38×22 최소) | 큼 (가변) |
| 콘텐츠 | **짧은 한 줄** | 긴 정보 + 액션 가능 |
| Title 폰트 | **11px Regular** | 15px Medium |
| 화살표 | 없음 | arrow Component Set |
| 사용자 액션 | **없음** (정보만) | 있음 (X 버튼 등) |
| 닫기 | 호버 해제 시 자동 | 외부 클릭 / X 버튼 |
| Padding | 8 / 4 (작음) | 12 / 9 (큼) |
| 강도 | **약함** (가벼움) | 중간 |

## 🧩 구조

```
Tooltip (radius 6, 가변 크기)
├── (선택) icon (12×12, 좌측)
├── Title (11px Regular)
└── (선택) icon (12×12, 우측)
```

> 🔑 매우 작은 컴포넌트 (최소 38×22) — **요소 위/아래에 살짝 떠있는** 인상

## 🎨 Variants — 1개 Component Set, **6 variants**

**2축 조합:**

| 축 | 옵션 |
|---|---|
| **Style** | `Black` / `White` / `Gray` |
| **Blur** | `off` (단색) / `on` (반투명·블러) |

### 6 Variants 상세

| Style | Blur | 배경 | Title 색 | 사용 |
|---|---|---|---|---|
| `Black` | `off` | `#3D3D3D` | white | **기본** — 일반 안내 |
| `Black` | `on` | 검정 70% | white | 콘텐츠 위 부드럽게 |
| `White` | `off` | white | `#3D3D3D` | 어두운 배경 위 |
| `White` | `on` | 흰색 70% | `#3D3D3D` | 어두운 배경 + 부드럽게 |
| `Gray` | `off` | `#8B8B8B` | white | 약한 강조 |
| `Gray` | `on` | 검정 40% | white | 가장 부드러운 표현 |

### Blur 차이

| | 효과 | 사용 |
|---|---|---|
| `off` | 단색 fill | 명확한 가독성 필요 |
| `on` | 반투명 + (앱에서 backdrop-filter blur 적용) | 콘텐츠 위 부드럽게 비춤 |

## 🎯 사용 토큰

### Tooltip 본체
- **Radius**: 6
- **너비/높이**: 가변 (콘텐츠에 맞춤, 최소 38×22)
- **Padding (좌우)**: 8
- **Padding (상하)**: 4

### 배경 (Style × Blur)
| Style | Blur=off | Blur=on |
|---|---|---|
| `Black` | `#3D3D3D` | `#000000` opacity 70% |
| `White` | `#FFFFFF` | `#FFFFFF` opacity 70% |
| `Gray` | `#8B8B8B` | `#000000` opacity 40% |

### Typography
- **Title**: 11px Regular

### Title 색
| Style | Color |
|---|---|
| Black / Gray | `static/white` |
| White | `label/strong` (`#3D3D3D`) |

### Icon (좌/우, optional)
- **사이즈**: 12 × 12
- **색**: Title 색과 동일
- 보통 비활성 (`visible: false`) — 필요 시 활성화

## 📖 사용 가이드

### Figma 가이드 원문 정리

> **원칙:** 요소의 **짧은 부가설명**이 필요할 때 사용합니다.
>
> **활용:**
> - 중요 콘텐츠를 가리지 않도록 **위나 아래에 위치**합니다.
> - **클릭이나 호버 등 사용자의 액션이 있을 경우** 안내를 제공합니다.

### 위치 가이드

> **"요소의 아래 또는 위에 사용합니다."** (Figma 사용예시 디스크립션)

```
[Tooltip 떠있음]
   ↓
[요소 (호버 대상)]
   ↑
[Tooltip 떠있음]
```

#### 위치 선택

| 상황 | 위치 |
|---|---|
| 요소 **상단**에 공간 충분 | **위쪽** (가장 일반적) |
| 요소 **하단**에 공간 충분, 위쪽 잘림 | **아래쪽** |
| 좌/우 — Tooltip은 권장 X | (Popover 사용) |

> 💡 **Tooltip은 위/아래만** — Popover와 달리 좌/우 배치는 일반적이지 않음

### 트리거 가이드

```
[데스크톱 — Hover]
요소에 마우스 올림 → 짧은 딜레이 (300~500ms) → Tooltip 등장
요소에서 마우스 떠남 → 즉시 또는 짧은 딜레이 → Tooltip 사라짐

[모바일 — Long press]
요소 길게 누름 → Tooltip 등장
손 뗌 → Tooltip 사라짐

[키보드 — Focus]
Tab 키로 포커스 이동 시 Tooltip 등장
포커스 해제 시 사라짐 (접근성!)
```

### Style 선택 가이드

```
일반적인 사용 (대부분의 케이스)
   → Style=Black, Blur=off
   → 가장 명확하고 가독성 좋음

이미지/복잡한 배경 위
   → Style=Black, Blur=on
   → 콘텐츠가 살짝 비치며 부드러운 인상

어두운 배경 (검정 콘텐츠 위)
   → Style=White, Blur=off
   → 흰 배경 + 검정 텍스트로 가독성

부드러운 보조 안내 (강조 약함)
   → Style=Gray (off 또는 on)
   → 너무 강하지 않은 정보 전달
```

### Blur 선택 가이드

| 상황 | 권장 |
|---|---|
| **단색 배경 위** | `Blur=off` (단색 더 명확) |
| **이미지/복잡한 배경 위** | `Blur=on` (콘텐츠와 자연스럽게 어우러짐) |
| **가독성이 가장 중요** | `Blur=off` |

### 텍스트 작성 가이드

| 좋은 예 | 나쁜 예 |
|---|---|
| "저장" | "이 버튼을 클릭하면 현재 작업 내용이 저장됩니다" (너무 김) |
| "Ctrl+S" | "저장 단축키" (Tooltip엔 단축키 자체가 더 효과적) |
| "권한 없음" | "이 작업을 수행하기에는 사용자 권한이 부족합니다" (너무 김) |

- **한 줄, 짧게** (5~20자 권장)
- **명사형** 또는 **짧은 명령형**
- 두 줄 이상이면 → **Popover** 사용

### 호버 딜레이 가이드 (UX)

```
나타날 때: 300~500ms (너무 빠르면 시선 어지러움)
사라질 때: 즉시 또는 100~200ms

연속된 요소 (예: 툴바 버튼들):
   첫 Tooltip 등장 후 → 다른 요소로 이동 시
   딜레이 없이 바로 새 Tooltip 등장 (자연스러운 흐름)
```

### 동시 노출

- **여러 Tooltip 동시 노출 X** — 한 번에 하나만
- 새 요소에 호버 시 기존 Tooltip 즉시 사라짐

### 접근성 (a11y)

- **Tab 포커스 시에도 Tooltip 노출** — 키보드 사용자 배려
- **`aria-describedby`** 또는 **`aria-label`** 속성 활용
- 스크린리더가 Tooltip 내용을 읽을 수 있도록

## 🔄 State 처리

### 흐름도

```
[Hidden]
   ↓ hover / focus 시작
[Showing] (300~500ms 딜레이)
   ↓
[Visible]
   ├ 다른 요소로 호버 이동 → 즉시 [Hidden]
   ├ 호버 해제 → [Hiding] → [Hidden]
   └ ESC 키 → 즉시 [Hidden]
[Hiding] (100~200ms)
   ↓
[Hidden]
```

### 등장/사라짐 애니메이션 권장

- **등장**: 페이드 인 + 살짝 슬라이드 (위에서 등장 시 위에서 아래로) — 150ms
- **사라짐**: 페이드 아웃 — 100ms

## 🚫 금지 사항

- ❌ **두 줄 이상 긴 텍스트** — 한 줄 권장, 길면 Popover
- ❌ **클릭 트리거** — Tooltip은 호버/포커스 전용 (Popover와 헷갈림)
- ❌ **사용자 액션 추가** (닫기 버튼, 링크 등) — 정보만 제공
- ❌ **여러 Tooltip 동시 노출** — 한 번에 하나만
- ❌ **좌/우 배치** — Tooltip은 위/아래만 (Popover는 가능)
- ❌ **너무 빠른 등장** (즉시 등장) — 의도치 않은 호버에도 뜸 → 300~500ms 딜레이 필요
- ❌ **너무 느린 등장** (1초+) — 사용자가 기다리다 시선 이탈
- ❌ **모바일에서 호버 트리거** — 모바일은 long press 또는 다른 패턴
- ❌ **중요 콘텐츠 가림** — 위치 자동 조정 (잘리면 반대편)
- ❌ **Tooltip 안에 또 다른 Tooltip** — 중첩 X
- ❌ **클릭 가능한 요소 표시** — Tooltip은 정보만, 클릭 필요하면 Popover
