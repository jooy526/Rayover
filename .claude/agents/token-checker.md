# Agent: Token Checker

Figma 토큰과 코드 토큰을 비교하고, 불일치를 보고하며 승인 시 자동 수정합니다.

> Figma는 항상 source of truth. 코드가 Figma를 따름.

## 트리거
- "토큰 확인해줘"
- "Figma랑 코드 색상 맞는지 확인"
- `token-checker` 명시 호출

## 프로세스

### 1. Figma 현재 값 수집
Figma MCP `get_variable_defs`로 현재 토큰 값 추출.

### 2. 코드 토큰 파일 확인
`docs/DESIGN.md` 및 CSS 변수 파일에서 현재 값 추출.

### 3. 불일치 보고
```
토큰 불일치 보고:
┌─────────────────────┬──────────────┬──────────────┐
│ 토큰                │ Figma 값     │ 코드 값      │
├─────────────────────┼──────────────┼──────────────┤
│ primary/normal      │ #FF7733      │ #FF6622 ❌   │
│ status/success      │ #00A371      │ #00A371 ✅   │
└─────────────────────┴──────────────┴──────────────┘
불일치 항목: 1개

자동 수정할까요? (승인 필요)
```

### 4. 승인 후 수정 (원칙 3: Surgical)
- 불일치 토큰만 수정
- 다른 코드 건드리지 않음
- 수정 후 diff 보고

## 검사 범위
- `docs/DESIGN.md` 색상 값
- CSS 변수 파일 (`*.css`, `tailwind.config.*`)
- 컴포넌트 inline style (하드코딩 감지)
